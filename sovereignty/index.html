Principles

Sovereignty

Sovereignty is the ability to operate intelligent systems without surrendering authority, data custody, or decision control to external entities.

Owner authority

Critical actions require explicit permission. No silent autonomy. No hidden side-effects.

Policy before capability

Tools and actions are gated by policy, risk tiers, and operator consent.

Audit by default

Intent → policy evaluation → execution → evidence. Traceability is a first-class output.

Non-negotiables

  • Local-first posture when possible: data stays under operator control.
  • Explicit side-effect boundaries: network, file writes, and external calls are gated.
  • Immutable governance primitives: the “rules” cannot be bypassed at runtime.
  • Evidence over vibes: logs, manifests, signatures, and reproducible traces.

What this looks like in practice

A sovereign assistant can brainstorm freely — but it cannot execute risky actions (send, delete, transmit, purchase, post, modify systems) without passing governance gates and obtaining operator consent.
